## 一、RAG 解决什么问题

### 1.1 大模型的局限性

ChatGPT 等大模型非常强大，但有三个关键限制：

- **知识截止**：训练数据有截止日期，无法回答最新信息
- **无法访问私有数据**：不知道你的公司文档、产品手册、内部规范
- **幻觉问题**：对不确定的内容可能"编造"看似合理的答案

### 1.2 RAG 的核心思路

**RAG（Retrieval-Augmented Generation，检索增强生成）** 的思路很直接：

> 不让大模型凭记忆回答，而是先从你的文档中找到相关内容，再让大模型基于这些内容来回答。

类比：开卷考试 vs 闭卷考试

| | 闭卷考试（纯 ChatGPT） | 开卷考试（RAG） |
|---|---|---|
| 知识来源 | 只靠记忆 | 可以翻书 |
| 准确性 | 可能记错 | 有据可查 |
| 适用范围 | 通用知识 | 特定领域 |
| 可信度 | 可能瞎编 | 有来源引用 |

### 1.3 典型应用场景

- 企业知识库问答（HR政策、产品手册、操作规范）
- 客服系统（基于FAQ和产品文档的智能回答）
- 法律/医疗/金融等专业领域的文档问答
- 代码库问答（基于项目文档回答技术问题）

## 二、与普通 ChatGPT 的区别

### 2.1 直观对比

**场景**：你上传了公司的《退货政策说明》，然后问："我买了7天的商品能退吗？"

**普通 ChatGPT 的回答**：

> 一般来说，大部分商家提供7天无理由退货服务...（基于通用知识回答，与你公司的实际政策无关）

**RAG 知识库的回答**：

> ✅ 以下回答基于知识库检索结果
>
> 根据退货政策第3条，购买7天内的商品支持无理由退货，但需满足以下条件：1. 商品未拆封... 2. 保留购物凭证...（来源：退货政策说明.pdf）

### 2.2 核心差异

| 维度 | ChatGPT | 本项目（RAG 知识库） |
|------|---------|---------------------|
| 数据来源 | 预训练数据 | 你上传的文档 |
| 回答依据 | 模型记忆 | 文档检索结果 |
| 是否可引用来源 | 否 | 是（显示来源文档、相似度、引用原文） |
| 能否回答私有问题 | 否 | 是 |
| 知识更新 | 需要重新训练 | 上传新文档即可 |
| 幻觉控制 | 无 | 严格要求基于检索内容回答，不相关时明确告知 |
| 对话历史 | 有 | 有（最近10轮对话上下文） |

## 三、核心功能演示

> 以下是项目的四个核心功能，演示顺序即完整的 RAG 使用流程。

### 3.1 功能一：知识库管理

**页面**：知识库列表页

**操作**：
1. 点击「新建知识库」
2. 输入名称（如"公司退货政策"）和描述
3. **配置 RAG 参数**：Embedding 模型、分块大小（默认500字）、重叠大小（默认50字）
4. 创建成功后，自动跳转到知识库详情

**要点**：
- 知识库是文档的容器，一个知识库可以包含多个文档
- 支持创建、**编辑**（可修改 RAG 配置参数）、删除、启用/禁用
- **RAG 配置可视化**：Embedding 模型、Chunk 大小、重叠大小都可以在前端直接看到和编辑
- 卡片式展示，显示文档数量和分片数量

### 3.2 功能二：文档上传与解析

**页面**：知识库详情页

**操作**：
1. 查看顶部 **RAG 配置栏**（显示当前知识库的 Embedding 模型、Chunk 大小、重叠大小）
2. 点击「上传文档」
3. 选择一个文件（支持 PDF、Word、TXT、MD 等格式，需 doc-parser 服务）
4. 观察文档状态变化：`PENDING → PARSING → CHUNKING → EMBEDDING → COMPLETED`

**要点**：
- 上传后触发异步处理：解析 → 分块 → 向量化
- 每个阶段都有明确的状态（失败时显示 PARSE_FAILED / CHUNK_FAILED / EMBEDDING_FAILED）
- **无降级方案**：doc-parser 不可用时直接报错，确保教学时每个环节都是真实的
- 已完成的文档可以点击「重新解析」重新走一遍 RAG 管道（例如修改了分块配置后）

### 3.3 功能三：切片查看

**页面**：文档切片页

**操作**：
1. 在文档列表中点击「查看切片」
2. 左侧：原文内容（高亮当前选中的切片）
3. 右侧：切片列表（每个切片是一段文本）

**要点**：
- 长文档被切成多个小片段（Chunk），大小由知识库配置决定（默认约 500 字）
- 切片是向量检索的基本单位
- 点击切片可以看到它在原文中的位置

### 3.4 功能四：智能问答

**页面**：智能对话页

**操作**：
1. 选择要查询的知识库（可以多选，也可以 x 掉取消关联）
2. 输入问题（如"退货需要什么条件？"）
3. 查看 AI 的回答：
   - 开头标注 ✅（基于知识库）或 ⚠️/❌（未使用/不相关）
   - 每段内容标注来源文档名
   - 底部显示**参考来源**（文档名 + 相似度 + 引用原文片段）
4. 连续提问，验证对话历史（AI 记住上下文）

**要点**：
- AI 回答严格基于知识库内容，不会幻觉编造
- 如果检索到的内容与问题不相关，AI 会明确告知（❌），不会强行回答
- x 掉知识库后再发消息 → 不走 RAG，纯 LLM 回答
- 支持对话历史，切换会话可以看到之前的聊天记录

## 六、对标产品

本项目是 **RAGFlow** 和 **Dify** 的教学简化版：

| 能力 | RAGFlow/Dify | 本项目 |
|------|-------------|--------|
| 文档解析 | 深度解析（OCR、表格、图片） | doc-parser 基础解析（无降级） |
| 分块策略 | 多种策略可配置 | 固定长度分块，参数可配置（前端可视化） |
| 向量存储 | Milvus/Weaviate 专业向量库 | 内存 ConcurrentHashMap（重启自动重建） |
| 检索策略 | 混合检索 + Rerank | 余弦相似度 + 相似度阈值 + 相关性判断 |
| 问答生成 | 流式输出 + 多轮对话 | 非流式 + 多轮对话（10轮历史） |
| 幻觉控制 | Prompt 约束 | 严格 Prompt 约束 + 相关性判断 + 来源标注 |
| RAG 配置 | 后台配置 | 前端可视化配置（Embedding 模型、Chunk 大小、重叠） |

> 教学项目聚焦于让学生理解 RAG 完整链路，每个环节都是真实的，没有降级和 mock。
