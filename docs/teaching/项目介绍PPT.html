Spring Boot 3 Vue 3 Python FastAPI RAG 全链路 OpenAI API
BACKGROUND
为什么做这个项目？
大语言模型虽然强大，但存在三个关键限制
📅
知识截止
训练数据有截止日期，无法回答最新发生的事情，比如最近的政策变更
🔒
私有数据盲区
不了解企业内部文档、产品手册、规章制度等私有知识
🌀
幻觉问题
对不确定的内容可能"编造"看似合理实则错误的答案，难以溯源
CORE IDEA
RAG — 检索增强生成
不让大模型凭记忆答题，而是先从文档中找到答案再让它组织回答
📝
闭卷考试（纯 ChatGPT）
只靠模型记忆回答，可能记错、可能编造、不能回答私有问题、无法引用来源
📖
开卷考试（本项目 RAG）
先检索文档找到相关段落，再基于原文生成回答，有据可查、来源可追溯、准确可控

HIGHLIGHTS
核心设计亮点
  • 1
    手写余弦相似度 — 没有使用向量数据库黑盒，自己实现 cosineSimilarity 算法，理解向量检索的数学本质
  • 2
    启动自动重建向量索引 — VectorIndexRebuilder 监听 ApplicationReadyEvent，重启后自动从 DB 重建内存向量，无需重新上传文档
  • 3
    三重防幻觉机制 — System Prompt 明确禁止编造 + 相关性判断（不相关时明确告知 ❌）+ 来源标注（每段引用真实文档名）
  • 4
    RAG 参数前端可配 — Embedding 模型、分块大小、重叠大小都可以在前端可视化编辑，修改后支持重新解析
  • 5
    真实依赖无降级 — doc-parser 不可用时直接报错而非降级，确保每个环节都是真实的外部调用
  • 6
    动态知识库选择 — 对话中可随时添加/移除关联知识库，空数组时切换为纯 LLM 模式

能力维度RAGFlow / Dify本项目

向量存储Milvus / Weaviate 专业向量库ConcurrentHashMap + 启动自动重建
检索策略混合检索 + Rerank手写余弦相似度 + 阈值过滤 0.3
幻觉控制Prompt 约束三重防幻觉 + 来源标注 + 相关性判断
对话能力流式输出 + 多轮多轮对话（10 轮历史）+ 动态 KB
